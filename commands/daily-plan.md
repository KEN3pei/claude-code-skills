# 今日の作業計画スキル

今日やりたいことを会話形式で洗い出し、優先順位と時間見積もりを整理して、スケジュール表を作成する。

---

## 全体の流れ

**Phase 1 → Phase 2 → Phase 3** の順で進める。各フェーズの詳細は以下の通り。

---

## Phase 1: タスク洗い出し

### 開始メッセージ

```
今日やりたいことを教えてください。思いついた順で構いません。
```

### 進め方

- ユーザーが挙げたタスクを都度リスト化して確認しながら進める
- タスクがざっくりすぎる場合（例：「作業する」「調べる」）は、以下のように具体化を促す
  - 「『調べる』というのは、具体的に何を調べますか？」
  - 「『作業する』はどのシステム・コンポーネントですか？」
  - 「アウトプットはどういう状態になれば完了ですか？」
- ユーザーが「以上」「終わり」「OK」「完了」「もうない」などと言ったらPhase 2へ移行する

---

## Phase 2: 順序・時間見積もり

### 開始時の質問（一度だけ）

```
何時から作業を始めますか？（例：10:00）
```

- 開始時刻を記録し、以降の開始時刻は累積で自動計算する

### 繰り返しの対話

以下のサイクルを繰り返す：

1. 「まず何から取りかかりますか？」（Phase 1で挙げたタスクの一覧を毎回表示する）
2. 「それはどのくらいかかりそうですか？（例：1h、30min）」
3. スケジュール済みリストと **現在の合計時間** を表示する

### 表示フォーマット（毎ターン）

```
📋 スケジュール済み（合計: X時間Y分 / 残り: Z時間W分）
1. タスクA（1h） → 10:00〜
2. タスクB（2h） → 11:00〜
...

残りタスク（未スケジュール）:
- タスクC
- タスクD
```

### 終了条件

- **合計が7時間に達した場合**: 「スケジュールが埋まりました！まとめますか？」と確認し、YesでPhase 3へ
- **ユーザーが「終わり」「完了」「以上」「OK」と言った場合**: 残り時間を表示してからPhase 3へ
  - 例：「了解です。残り X 時間分は空きとなります。スケジュールをまとめます。」

---

## Phase 3: スケジュール出力

以下のマークダウンテーブル形式で出力する。

```markdown
## 今日のスケジュール

| 順番 | タスク名 | 見積もり時間 | 開始時刻 |
|------|---------|------------|---------|
| 1    | タスクA  | 1h         | 10:00   |
| 2    | タスクB  | 2h         | 11:00   |
| 3    | タスクC  | 1.5h       | 13:00   |

合計: X時間 / 空き: Y時間
```

---

## 補足指示

- すべて日本語で会話する
- 時間の表記は「1h」「30min」「1.5h」など柔軟に受け付け、内部では分単位で管理する
- 開始時刻の計算は前のタスクの終了時刻を次の開始時刻とする（休憩は考慮しない、ユーザーが「休憩30分」と言った場合はタスクとして追加する）
- Phase 2でユーザーが時間を「わからない」と言った場合は「30分・1時間・2時間のどれが近いですか？」と選択肢を出す
