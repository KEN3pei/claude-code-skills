# X ブックマーク技術解説プロンプト

## 引数

```
/analyze-bookmarks <bookmark-file-path>
```

引数として渡されたファイルパス: `$ARGUMENTS`

---

## Claudeへの指示

`$ARGUMENTS` のパスにあるファイルをReadツールで読み込み、以下の手順で処理してチャット上に返答する。

### ステップ1: ファイルの読み込み

`$ARGUMENTS` のMarkdownファイルを読み込み、各エントリ（`## N.` で区切られたブロック）を順番に処理する。

### ステップ2: リンク先の取得

各ツイートの本文に URL（`https://t.co/...` や その他リンク）が含まれる場合、WebFetchツールでリンク先を取得する。

- `https://x.com/` または `https://twitter.com/` で始まるURL → スキップ（認証が必要なため取得不可）
- 取得できない場合（タイムアウト・404・認証エラー等）→ 「リンク先取得不可」と明記してツイート本文のみで分析
- 取得できた場合 → その内容をステップ3の分析に活用する

### ステップ3: 各ツイートの分析

各ツイートについて以下を実行する（リンク先が取得できた場合はその内容も踏まえて分析する）：

**A. トピック特定**
- 何の技術・話題・概念について言及しているかを1〜3語で特定する
- 例: `JWT認証`, `NGINX脆弱性`, `Goコネクションプール`, `XSS対策`

**B. 深掘り判定**（以下のいずれかに該当すれば「深掘り」対象）
- 技術的な概念・仕組み・アルゴリズムが含まれる
- セキュリティ脆弱性・CVEへの言及がある
- ツール・ライブラリ・プロトコルの紹介がある
- 設計パターン・アーキテクチャ上の判断の話をしている
- 「なぜこうなるのか」の原因・背景が含まれる

該当しない場合（感想のみ・リンクのシェアのみ・近況報告のみ）は「一言サマリ」扱い。
ただしリンク先を取得した結果、技術的な内容だった場合は「深掘り」対象として扱う。

**C. 解説生成**

深掘り対象の場合：
```
【トピック】<1〜3語>
【概要】<何についての話か・2〜3文>
【技術解説】<仕組み・背景・原因を詳しく説明。必要に応じて箇条書きや対比を使う>
【エンジニア的注目点】<なぜ気にすべきか・実務でどう関係するか・1〜3文>
```

一言サマリの場合：
```
【トピック】<1〜3語>
【概要】<ひとことで何のツイートか>
```

### ステップ4: チャットに出力

以下の構造でチャット上に返答する。

```
# X ブックマーク解説 - YYYY/MM/DD

> 合計N件 / 深掘り対象M件

---

## 1. <トピック名>
投稿日時: YYYY/MM/DD
URL: <ツイートURL>

（深掘りの場合）
**概要**: ...
**技術解説**: ...
**エンジニア的注目点**: ...

（一言の場合）
**概要**: ...

---

## 2. ...
```

---

## 補足指示

- 解説はすべて日本語で書く
- 英語の技術用語はそのまま使ってよい（例: JWT, NGINX, cgroup, eBPF）
- 「おそらく」「推測ですが」など、確信が持てない解説には必ずその旨を明記する
- ツイート本文が途中で切れている場合は、切れている部分について推測で補足せず「本文が途中で切れています」と記載する
- Web検索が必要な場合は検索して情報源URLを添える
